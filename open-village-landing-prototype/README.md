# Open Village Landing Prototype

Standalone-прототип лендинга для выставки Open Village для компании D² DOM / ДеревДом с фокусом на отделку домов и квартир.

## Что внутри

- `index.html` — готовая структура лендинга по секциям.
- `css/style.css` — весь визуальный слой, адаптив и namespaced-классы `.ov-*`.
- `js/main.js` — логика многошагового мини-квиза без backend и без реальной отправки формы.
- `assets/` — локальные шрифты D² DOM и SVG-заглушки для проектов, команды и QR-кода.

## Структура страницы

1. Hero screen с двумя CTA и зоной под QR-код.
2. Блок "Для кого этот лендинг".
3. Блок услуг по отделке и ремонту.
4. Блок преимуществ цифрами.
5. Блок проектов на 6 карточек.
6. Акцентный блок для гостей Open Village.
7. Мини-квиз на 4 шага с финальным экраном благодарности.
8. Блок команды.
9. Финальный CTA.

## Как открыть прототип

1. Откройте файл `index.html` в браузере напрямую.
2. Либо поднимите любой простой локальный static server и откройте папку `open-village-landing-prototype`.

Пример:

```bash
cd open-village-landing-prototype
python3 -m http.server 8000
```

После этого откройте `http://localhost:8000`.

## Что заменить перед публикацией

- реальные фотографии проектов и команды вместо placeholder SVG;
- реальный QR-код;
- реальные карточки проектов: названия, площади, сроки, описания, локации;
- реальные контакты и/или интеграцию формы;
- обработку отправки квиза, аналитику и CRM-интеграцию;
- финальные юридические тексты, если форма будет собирать персональные данные.

## Технические примечания

- Вёрстка сделана без framework и без backend.
- Все CTA ведут к блоку квиза `#ov-quiz`.
- Для интеграции использован изолированный namespace классов `.ov-*`, чтобы минимизировать конфликты.
- Визуальный стиль собран в светлой теме с локальными шрифтами и без внешних библиотек.
- Смысловой слой страницы адаптирован под позиционирование D² DOM на выставке как компании по отделке домов и квартир, без акцента на строительство.
